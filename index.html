HTML

  • The head element contains encoding information about the webpage for browsers
    • Viewport allows content to be scaled properly for display across devices
  • The body element contains content visible to the user
  • When using headers, always start with h1 before moving to subsequent header numbers, and only have h1 appear once per page; this is for better optmization with assistive technology
  • "src" = "source" of an image, etc.
  • To comment out, type the following without spaces: < !--code you want commented out-- >
  • Using elements in HTML to make text italic, bold, and more: visit this helpful blog page, which also includes information on how to accomplish the same in CSS when more appropriate
  • To insert a link, type without spaces: < a href:"URL"> text you want to display as the link < /a>

CSS

  • Cascading Style Sheets, or CSS, is a stylesheet language used to style an HTML document by describing how HTML elements should be displayed
  • Margin: how much space will be around the outside of an element
  • Padding: how much space will be between the content inside an element
  • For basic info on CSS, visit MDN Web Docs' article on CSS
  • For a tutorial with tons of examples, visit W3 Schools' CSS Tutorial webpage
  • For information on using properties in CSS to emphasize text, visit this helpful blog page from the HTML notes section above

Git

  • The general flow of using GitHub:
    • Create repo in GitHub
    • Clone to local machine (git clone [repo name])
    • Create new working branch with descriptive name pertaining to changes being made (git checkout -b [new branch name])
    • Make desired changes
    • Stage desired changes (git add -A to stage all)
    • Commit desired changes with a comment, ideally in imperative present tense, describing changes being committed (git commit -m "[comment here])
    • Ensure active branch is up to date before making PR or pushing changes with git pull origin main
    • Push changes with git push origin [active branch name]
    • Make pull request in GitHub comparing main as base to new ("comparison") branch with commits to merge
    • Merge comparison branch into main branch
    • Close corresponding issue in GitHub if resolved by changes
    • Update local copy of main branch (git checkout main)
    • Pull remote main branch into local main branch (git pull origin main)
    • Make new branch for additional changes, repeat process as necessary
  • Git terminal commands and info:
    • git pull [branch name] (e.g., git pull origin main): pulls specified branch to update local repo
    • git status: checks status of changes made, lists staged and ustaged changes
    • git log: lists history of commits made in descending chronological order
    • git branch: lists branches and indicates active branch with * followed by name of branch
    • git checkout [branch name]: switches to specified branch
    • git checkout -b [branch name]: creates new branch and makes it active
    • git --amend: amend most recent commit's comment/message (useful for correcting typos or other issues with commit messages)
    • git push [remote branch to update] [active branch to update with]: pushes active branch's changes to specified remote branch (usually origin)
      • Info on git push commands here
    • Merging vs. rebasing: Link to Atlassian git tutorial on merging vs. rebasing
    • git merge [working branch name] [target branch name]: merges target branch into working branch (e.g., git merge feature main would merge main branch into feature branch, creating a new merge commit in the feature branch tying the histories of both together
      • git merge can also be used to incorporate changes from specified commits since the point their histories diverged from the active branch into the active branch
        • Running git merge when "non-trivial" changes have been made is discouraged: it can cause merge conflicts that are difficult to resolve
      • More information on uses of git merge here
    • Squashing commits:
      • In some cases, it might be helpful to "squash" the number of commits down to one commit when going to merge your work back into the main branch. To squash commits, take the following steps:
        • Start an interactive rebase session with git rebase -i HEAD~[number of most recent commits you'd like to squash]
        • The number of commits will show up in the editor preceded by "pick." Replace "pick" with "squash."
        • Enter a message for the new, combining commit, completing the interactive rebase.

JavaScript

JavaScript is an object-oriented programming language, meaning that it organizes software design around data (objects) instead of functions and logic.

  • Control flow
    • The order a computer executes code (from the first line to the last line)
    • Using conditional statements and loops can change order code is run or prevent code from running at all
  • Object
    • A data field/item with a unique attribute and behavior
    • A value in memory which is possibly referenced by an identifier
  • Identifier
    • A sequence of characters in code that identifies a variable, function, or property
    • JavaScript identifiers are case-sensitive and can contain Unicode letters, $, _, and digits 0 through 9, but can't start with a digit
  • Variable
    • A named container allowing you to store data in your code
  • Array
    • An object storing more than one kind of data, usually related to each other, under one variable name
    • Grouping data in an array is a fundamental feature of JavaScript
  • Property
    • A characteristic of an object, often describing attributes associated with a data structure
    • Instance properties hold data specific to a given object instance
    • Static properties hold data shared between all object instances
    • A property has a name (a string or symbol) and value (primitive, method, or object reference)
  • Primitive values
    • Immutable (i.e., unchangeable) data represented directly at the lowest level of the language
      • Boolean: Can have value of True or False
      • Null: Has value of null
      • Undefined: A variable that hasn't been assigned a value
      • Numeric types: Number, BigInt, and NaN
        • Number: A numeric value in the IEEE 754 format
        • BigInt: A numeric primitive that can represent integers outside the "safe" limit, created by adding n to the end of an integer or by calling the constructor
        • NaN: "Not a Number"
      • String: Textual data; a set of "elements" of 16-bit unsigned integer values with each occupying a position in the string
        • As a zero-indexed, sequential language, the first element in a JavaScript string will always be at index 0, the next at 1, and etc.
        • Must always be enclosed in quote marks " "
          • Example: var StudentInfo = ["Kate", 28, true];
      • Symbol: A unqiue, immutable primitive value that can be used as the key of an object property; sometimes called "atoms" in other languages
  • Constructor
    • A method of a class for making and initializing an object instance of that class
    • Allows you to provide any custom initialization that must be done before any other methods can be called on an instantiated object
    • See MDN Web Docs' page on constructors for more info
  • Index
    • A unique numerical indentifier automatically assigned to every item in an array, starting from 0 (see notes on strings above)
    • Always a whole number
    • The order an element appears in a an array or string determines its index
  • Method
    • A set of instructions that a computer can execute
  • Commenting out: /* [...] */ and //

Vim

Vim, included as "vi" with most Unix systems and OS X, is an open-source text editor designed for using from the terminal or as a standalone text editor with a GUI. It has 12 different editing modes.

  • Modes:
    • Normal: The default mode, used for editor commands; hit Esc to enter Normal mode
    • Visual: Used to highlight areas of text to run normal commands on
    • Insert: Allows editing/insertion of text; hit i to enter Insert mode
    • Command-line / Cmdline: Allows a single line input at the bottom of the Vim window; upon completion of the command, Vim returns to the previous mode
  • To navigate lines of text, use the h, j, k, and l keys instead of the mouse or arrow keys
  • To navigate between words, use the w, b, and e keys
  • You can combine movement keys with a number to move the cursor as instructed multiple times (e.g., 3w is the same as hitting w three times)
  • For an interactive Vim tutorial, visit OpenVim.com
  • For a very helpful cheatsheet on Vim shortcuts and commands, take a look at this online cheat sheet.
