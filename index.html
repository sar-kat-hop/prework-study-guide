HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Viewport (in the head element) affects scaling for devices by size.
  • When using headers, always start with h1 before moving to subsequent header numbers, and only have h1 appear once per page. This is for better optmization with assistive technology.
  • "src" = "source" of an image, and potentially other files.
  • "alt" = alternative descriptive text, for assistive technology and SEO.
  • To comment out in HTML, use < ! -- [content you want commented out] -- > without spaces.
  • A margin indicates how much space will be around the outside of an element
  • Padding indicates how much space will be between the content inside an element

Git

  • git pull branch-name (e.g., git pull origin main): pulls specified branch to update local repo
  • git status: checks status of changes made, lists staged and ustaged changes
  • git log: lists history of commits made in chronological order
  • git branch: lists branches and indicates active branch with * followed by name of branch
  • git checkout branch-name: switches to specified branch
  • git checkout -b branch-name: creates new branch and makes it active
  • git --amend: amend most recent commit's comment/message (useful for correcting typos or other issues with commit messages)
  • git merge:
  • git push:
  • merging vs. rebasing: https://www.atlassian.com/git/tutorials/merging-vs-rebasing
  • git merge working-branch target-branch: merges target branch into working branch (e.g., git merge feature main would merge main branch into feature branch, creating a new merge commit in the feature branch tying the histories of both together.
  • In some cases, it might be helpful to "squash" the number of commits down to one commit when going to merge your work back into the main branch. To squash commits, take the following steps:
    • Start an interactive rebase session with git rebase -i HEAD~[number of most recent commits you'd like to squash]
    • The number of commits will show up in the editor preceded by "pick." Replace "pick" with "squash."
    • Enter a message for the new, combining commit, completing the interactive rebase.
